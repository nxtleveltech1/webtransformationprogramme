The defining question — the programme's master risk (RSK-001, Very High)

Is this a content migration, a web transformation, or a bounded hybrid?

"Several delivery streams imply transformation — OMDS adoption, V2 templates, IA change, analytics, observability, automated testing, revised publishing workflow, decommissioning — while the programme is described as a 'migration.'"

This must be formally resolved (DEC-001 / QST-001 / QST-016) because it changes scope, effort, budget, timeline and risk profile. Most downstream decisions stay Proposed until it lands. Day 2 action ACT-001 / ACT-028 carries the choice:

A · Tactical
Achievable in the window
Navigation-only across all sites + secure web IA (the agreed MVP, DEC-003). Lands meaningful value by end-Nov; defers uplift. Validated by ASM-010.
B · Bounded hybrid
Window with hard trade-offs
Product pages → V2 templates; everything else moves as-is/restyled (DEC-008). Requires the "as-is" definition (ACT-004) and ruthless prioritisation by success metric (ACT-018).
C · Full transformation
~end-2027
Full OMDS, V2, IA, analytics, testing, decommission. Highest value, but incompatible with the fixed end-Nov budget window (RSK-019).

1
Scope decision (DEC-001 / ACT-001)

Design foundations + "as-is" definition (DEC-006 / ACT-004)
IA, navigation, templates, content standards; and a formal "as-is" definition — blocks estimates, testing scope and decommission.
3
Component / DS V2 build (RSK-008 / DEP-003 / DEP-004)
Most components don't exist in Contentstack and must be built; DS V2 must reach Beta to unlock templates and Find an Advisor.
4
URL audit & redirects (ACT-017 / DEP-005)
~95% of URLs changing — must map current→new IA before switch-off; notify e-commerce (Nthabi) to protect GA tracking (RSK-021).
5
Contentstack dev → publishing (DEP-001)
Publishing can only start once Contentstack dev is complete and content / assets / URLs are signed off.
6
SEO engagement ≥ 2 months (DEC-011 / DEP-009)
SEO team needs ≥2 months lead before QA for tagging and validation — a long pole against the window.
7
Testing → Go-live readiness (DEP-002 / ACT-013)
Testing only begins once all pages are built in Beta; then ServiceNow API updates, security pen-test, comms and decommission must clear (RSK-013/022).
