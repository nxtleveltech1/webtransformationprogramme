## 2. Detailed discussion notes (new content)

### 2.1 EasiPlus quote-engine / "side drawer" mechanics
- André + Gareth discussed the EasiPlus page as a **unique "side drawer"** entry point: it runs the quote/calc inline (rates table behind it: e.g. R10,000 → premium), unlike other journeys where clicking **buy** sends the user into a separate buy chain (TFSA, Flexi, RA, etc.).
- It has bespoke copy throughout the form/page — flagged as a one-off that needs explicit handling. → relates to ACT-024/ACT-051 (calculator/EasiPlus scope) and QST-018 (TFSA template).

### 2.2 Scope definition tension & delivery options (key)
- Gareth re-emphasised the **definition of what we're doing**: if it were *purely* migration, "migration would have been done" — but the **new feature/module scope** adds impact going forward (e.g. onboarding a new client/tenant changes integration).
- **Options framing:** consider doing **navigation only** across all sites as an iterative/tactical implementation to hit the timeline; "tactical versus strategic" view.
- Stark scope signal: if the full discussed scope is taken to stakeholders, "you're probably looking at **the end of 2027**"; a **six-month** deliverable requires a reduced/tactical scope. → strengthens RSK-001; adds QST-016 (navigation-only option) and RSK-019 (timeline-vs-scope mismatch).
- Value of the exercise: it doesn't solve the problem but **shows how big the problem is**, enabling realistic stakeholder conversations.

### 2.3 Team-health reflections
- **Knowledge-system gap:** much of the delay is attributed to the lack of a **centralised, shared knowledge system / documentation** (history, relationships, how things work). → RSK-020, ACT-054.
- **Publishing volume concern (Natalie):** apprehension about volume with **one person covering publishing** — even a small change (e.g. adding an ad) means touching every page; the end-loaded timeline is worrying. → reinforces RSK-002, ISS-007.
- **Coordination concern:** in-stream work is understood, but **cross-stream coordination** remains the "grey area"; request for the final list of ~10 components to start on. → ACT-007 / RSK-012.
- Gareth stressed **tomorrow (Day 2)** is about what is **achievable**, **team health**, and preparing **stakeholder conversations**.
